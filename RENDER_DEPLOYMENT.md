# Render Deployment Guide for CredNXT

This guide explains how to deploy the CredNXT P2P lending platform to Render.com with automatic GitHub integration.

## Prerequisites

1. GitHub account with the CredNXT repository
2. Render.com account
3. PostgreSQL database (Render provides managed PostgreSQL)

## Quick Deploy

### Option 1: Using render.yaml (Recommended)

1. **Fork/Clone Repository**: Ensure your code is in a GitHub repository
2. **Connect to Render**: 
   - Go to [Render Dashboard](https://dashboard.render.com)
   - Click "New" → "Blueprint"
   - Connect your GitHub repository
   - Render will automatically detect `render.yaml` and set up:
     - Web service with auto-scaling
     - PostgreSQL database
     - Environment variables

### Option 2: Manual Setup

1. **Create Database**:
   - Go to Render Dashboard → "New" → "PostgreSQL"
   - Name: `crednxt-db`
   - Plan: Start with "Free" tier for testing

2. **Create Web Service**:
   - Go to Render Dashboard → "New" → "Web Service"
   - Connect your GitHub repository
   - Configure:
     - **Build Command**: `./render-build.sh`
     - **Start Command**: `npm start`
     - **Health Check Path**: `/api/health`

3. **Set Environment Variables**:
   ```
   NODE_ENV=production
   PORT=10000
   DATABASE_URL=[Auto-generated from database]
   JWT_SECRET=[Generate secure random string]
   SESSION_SECRET=[Generate secure random string]
   ```

## Environment Variables

| Variable | Description | Required | Auto-Generated |
|----------|-------------|----------|----------------|
| `NODE_ENV` | Environment mode | Yes | Set to `production` |
| `PORT` | Server port | Yes | Set to `10000` |
| `DATABASE_URL` | PostgreSQL connection string | Yes | From Render database |
| `JWT_SECRET` | JWT signing secret | Yes | Generated by Render |
| `SESSION_SECRET` | Express session secret | Yes | Generated by Render |

## Health Check Endpoints

The application provides multiple health check endpoints for monitoring:

- **`/api/health`** - Basic health check with database connectivity
- **`/api/ready`** - Readiness probe for traffic serving
- **`/api/live`** - Liveness probe for process health
- **`/api/health/detailed`** - Comprehensive health information

## Build Process

The build process (`render-build.sh`) performs:

1. **Dependency Installation**: `npm ci` for production dependencies
2. **Application Build**: 
   - Frontend build with Vite (`client/` → `dist/public/`)
   - Backend build with esbuild (`server/` → `dist/`)
3. **Database Migration**: `npm run db:push` (if DATABASE_URL is set)

## Troubleshooting

### Common Issues

**Build Failures:**
- Check build logs in Render dashboard
- Verify all dependencies are in `package.json`
- Ensure TypeScript compiles without errors

**Module Not Found Errors (ERR_MODULE_NOT_FOUND):**
- This typically occurs when dev dependencies are bundled into production
- The build scripts are configured to exclude Vite and React plugins
- Use the production-specific `index.prod.js` entry point
- Server build excludes: `vite`, `@vitejs/plugin-react`, Replit plugins

**Database Connection:**
- Verify `DATABASE_URL` is set correctly
- Check database service status in Render
- Review health check endpoint `/api/health`

**Environment Variables:**
- Ensure all required variables are set
- Check `/api/health/detailed` for missing variables
- Verify secrets are generated properly

### Build Process Details

The build uses separate scripts to avoid dev dependency conflicts:
- `build-client.sh`: Builds React frontend with Vite
- `build-server.sh`: Builds Express backend with esbuild, excluding dev tools
- `server/index.prod.ts`: Production server without Vite middleware

## Scaling

### Free Tier Limitations
- Web service sleeps after 15 minutes of inactivity
- 750 hours per month compute time
- 1 GB RAM, 0.1 CPU

### Paid Tier Benefits
- No sleep/downtime
- Auto-scaling based on traffic
- More compute resources
- SLA guarantees

## Custom Domain

To use a custom domain:
1. Go to your web service settings in Render
2. Add custom domain under "Custom Domains"
3. Configure DNS records as instructed
4. SSL certificates are automatically provisioned
